## Общая структура репозитория

Репозиторий организован следующим образом:

```
frontend-examples/
├── README.md                 # Главный README репозитория
├── PROJECT_GUIDELINES.md     # Данное руководство
├── project-name-1/           # Папка проекта 1
├── project-name-2/           # Папка проекта 2
└── ...
```

Каждый проект размещается в отдельной папке в корне репозитория.

## Требования к проекту

### Независимость

- Каждый проект должен быть **полностью независимым** и самодостаточным
- Проект должен запускаться без необходимости установки зависимостей из других проектов репозитория
- Все необходимые зависимости должны быть указаны в `package.json` проекта

### Минимализм

- Проект должен демонстрировать **одну конкретную фичу** или **набор тесно связанных фич**
- Избегайте добавления лишнего функционала, не относящегося к демонстрируемой теме
- Код должен быть максимально простым и понятным

### Актуальность

- Используйте актуальные версии библиотек и фреймворков
- Обновляйте проекты при выходе критических обновлений безопасности

## Структура проекта

Рекомендуемая структура проекта:

```
project-name/
├── README.md           # Обязательно: описание проекта
├── package.json        # Зависимости и скрипты
├── tsconfig.json       # Конфигурация TypeScript (если используется)
├── .env.example        # Пример переменных окружения (если требуются)
├── src/                # Исходный код
│   ├── components/     # React компоненты (если применимо)
│   ├── hooks/          # Кастомные хуки (если применимо)
│   ├── utils/          # Утилиты и хелперы
│   └── ...
├── public/             # Статические файлы
└── ...
```

## README проекта

Каждый проект **обязательно** должен содержать файл `README.md` со следующими разделами:

### Обязательные разделы

1. **Название и краткое описание**
   - Что демонстрирует данный проект
   - Какую проблему решает

2. **Технологии**
   - Список используемых технологий и библиотек

3. **Установка и запуск**
   - Пошаговая инструкция по установке зависимостей
   - Команды для запуска проекта

4. **Описание реализации**
   - Краткое объяснение ключевых моментов реализации
   - Ссылки на основные файлы с кодом

### Опциональные разделы

- **Переменные окружения** — если проект требует настройки
- **Примеры использования** — скриншоты или GIF-анимации работы
- **Известные ограничения** — если есть
- **Полезные ссылки** — документация, статьи по теме

### Пример README

```markdown
# Название проекта

Краткое описание того, что демонстрирует проект.

## Технологии

- React 18
- TypeScript
- Vite
- Другие библиотеки...

## Установка и запуск

1. Перейдите в папку проекта:
   ```bash
   cd project-name
   ```

2. Установите зависимости:
   ```bash
   npm install
   ```

3. Запустите проект:
   ```bash
   npm run dev
   ```

## Описание реализации

Объяснение ключевых моментов реализации...

## Полезные ссылки

- [Ссылка на документацию](https://example.com)
```

## Именование

### Папки проектов

- Используйте **kebab-case** для названий папок
- Название должно отражать суть демонстрируемой фичи
- Примеры: `web3-wallet-connection`, `sentry-setup`, `solana-integration`

### Файлы

- Компоненты React: **PascalCase** (например, `WalletConnect.tsx`)
- Утилиты и хуки: **camelCase** (например, `useWallet.ts`, `formatAddress.ts`)
- Конфигурационные файлы: стандартные названия (`vite.config.ts`, `tsconfig.json`)

### Переменные и функции

- Используйте **camelCase** для переменных и функций
- Используйте **PascalCase** для компонентов и классов
- Используйте **SCREAMING_SNAKE_CASE** для констант

## Код и стиль

### Общие рекомендации

- Предпочтительно использовать **TypeScript**
- Следуйте принципам чистого кода
- Добавляйте комментарии к сложным участкам кода
- Избегайте `any` типа в TypeScript

### Форматирование

- Используйте **Prettier** для форматирования кода
- Рекомендуемые настройки:
  ```json
  {
    "semi": true,
    "singleQuote": true,
    "tabWidth": 2,
    "trailingComma": "all"
  }
  ```

### Линтинг

- Используйте **ESLint** для проверки качества кода
- Рекомендуется использовать конфигурацию из шаблона Vite или Next.js

## Зависимости

### package.json

- Используйте `^` для минорных обновлений обычных зависимостей
- Для критичных зависимостей (например, фреймворков) можно указывать **точные версии**
- Разделяйте `dependencies` и `devDependencies`
- Добавляйте описание проекта в поле `description`

### Скрипты

Рекомендуемый минимальный набор скриптов:

```json
{
  "scripts": {
    "dev": "команда для запуска в режиме разработки",
    "build": "команда для сборки проекта",
    "preview": "команда для просмотра собранного проекта"
  }
}
```

### Переменные окружения

- Не храните секретные ключи в репозитории
- Создавайте файл `.env.example` с примерами переменных
- Добавьте `.env` в `.gitignore`

- [ ] Проект запускается локально без ошибок
- [ ] Код отформатирован и проходит линтинг

- [ ] Отсутствуют секретные ключи и токены в коде
